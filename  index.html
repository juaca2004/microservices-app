01 — Infraestructura

Arquitectura Multi-Región

Infraestructura desplegada en dos regiones de Azure con alta disponibilidad gestionada por Traffic Manager. Cada región es completamente independiente con su propio AKS, networking y stack de observabilidad.

West US 3

02 — CI/CD

Flujo GitOps

Pipeline automatizado con GitHub Actions para el despliegue de infraestructura y aplicaciones. ArgoCD mantiene el estado del cluster sincronizado con el repositorio Git en todo momento. El equipo adoptó GitHub Flow como estrategia de ramificación: cada funcionalidad se desarrolla en una rama corta, se revisa mediante Pull Request y se fusiona directamente a main, disparando el pipeline de despliegue automáticamente.

STEP 01

Push a Rama

Developer hace push y dispara el workflow de GitHub Actions

STEP 02

Azure Login

Autenticación con Service Principal e inyección de secretos desde Key Vault

STEP 03

Infra Deploy

Linkerd, Nginx, cert-manager, monitoring stack via Helm

STEP 04

ArgoCD Sync

ArgoCD detecta cambios y sincroniza los microservicios automáticamente

STEP 05

Multi-Región

Matrix strategy ejecuta el deploy en parallel en eastus2 y westus3

# GitHub Actions — helm-deploy.yml
 
strategy:
  matrix:
    region:
      - name: eastus2 | aks_cluster: aks-dev-cluster_r1
      - name: westus3 | aks_cluster: aks-dev-cluster_r2
 
✓ Checkout helm repo
✓ Azure login (Key Vault secrets)
✓ Install Linkerd + tracing
✓ Deploy monitoring stack
✓ Deploy microservices via ArgoCD

04 — Seguridad

Capas de Seguridad

Seguridad implementada en múltiples capas: desde la gestión de identidades y secretos hasta el cifrado de tráfico entre microservicios.

mTLS Automático

Linkerd cifra automáticamente todo el tráfico entre microservicios con mTLS sin modificar el código de las aplicaciones.

Azure Key Vault

Secretos almacenados en Key Vault por región. El pipeline los crea y recupera dinámicamente enmascarando los valores en los logs.

OAuth2 Proxy

Acceso al dashboard de Linkerd protegido con OAuth2 Proxy integrado con Azure AD para autenticación centralizada.

Cert-Manager + GoDaddy

Certificados TLS automáticos con cert-manager y webhook de GoDaddy para validación DNS, renovación automática sin downtime.

Service Principal con RBAC

Dos Service Principals separados: uno para infraestructura (Terraform) y otro para el despliegue de aplicaciones, con permisos mínimos.

Linkerd Policy Auth

Políticas de autorización de Linkerd para controlar qué servicios pueden comunicarse entre sí dentro del mesh.

05 — Calidad & Resiliencia

Pruebas de Testing

Validamos el rendimiento y la resiliencia de la plataforma mediante pruebas de carga con Locust e ingeniería del caos con Chaos Mesh y Litmus, garantizando que el sistema aguanta condiciones extremas.

⚡

Pruebas de Carga — Locust

Se ejecutaron dos escenarios de prueba sobre los microservicios expuestos para medir el comportamiento bajo carga sostenida y ante picos repentinos de tráfico.

Locust Python HTTP
🔁 Prueba 1 — Carga Sostenida

Simulación de usuarios concurrentes incrementales para medir throughput, latencia promedio y tasa de errores bajo condiciones normales de operación.

⚡ Prueba 2 — Spike

Incremento abrupto de tráfico para evaluar la capacidad del sistema de escalar rápidamente y recuperarse sin degradación del servicio.

🔥

Chaos Engineering — Litmus

Se aplicó ingeniería del caos con Litmus sobre el clúster AKS para validar la tolerancia a fallos, la recuperación automática y la resiliencia del service mesh.
